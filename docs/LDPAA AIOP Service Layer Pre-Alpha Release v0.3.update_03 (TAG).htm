LDPAA AIOP SL V0.3.update_03 (TAG)

  

 

Hi,

 

We are pleased to announce the Pre-Alpha release v0.3.update_03 (TAG) of AIOP Service Layer.

 

The release is done from the ldpaa/aiopsl GIT and is tagged: ldpaa-aiop-sl-v0.3.update_03.

 

Please, refer to the GIT ChangeLog for additional details.

 

Report bugs on Clear Quest to the following BINs: LS-AIOP-LOW-LEVEL and LS-ARENA.

 

The main additions/changes in this version are:

 

·         AIOP initialization now configures the number of tasks per core to 16.

·         The __SHRAM attribute can be used to place global variables in the Shared Ram.  

·         The __HOT_CODE attribute can be used to place code in the iRAM.

·         Several minor changes were made to the SLAB API. 

·         The Event Manager module has been removed.

·         Added the following FDMA SR functions: fdma_close_segment(), fdma_delete_segment_data() and fdma_discard_fd().

·         Added fdma_terminate_task() at the end of function receive_cb().

·         The CDMA functions were updated according to ArchDef 0.7.5:

o   The size was removed from the acquire command.

o   The refcount was increased to 32 bits.

·         Updated some intrisic functions due to compiler 275 upgrade.

·         Changed implementation to use ARENA's function slab_find_and_fill_bpid which is used currently only by Parser and KeyGen pool ID creation.

·         Added functions aiop_sl_init() and aiop_sl_free().

·         Updated some KeyGen defines in order to comply with CTLU spec 0.7.2.

·         Fixed ENGR310243 (KeyGen related).

·         Added mask to the return status of the tman timer query function.

·         Fixed in Timer query return status codes.

 

 

Since this release is only a TAG release we did not attach any new release notes. These are the additions to the previous release notes (v0.3):

 

Ø  In order for IPR FM to work properly:

User should initialize 1 pool of the BMAN ,currently pool_id=1 using buffer size of at least 2632 bytes.

The number of buffers that should be initialized, is equal to the number of concurrent reassembled frames plus number of instances..

 

Ø  Fields changes in lcf files:

Those fields are needed to support __SHRAM and need to be placed in shared ram GROUP in lcf file

.shdata (DATA) : {}

.shbss (BSS) NO_INIT_BSS : {}

 

For the __HOT_CODE the following fields should be placed in iram GROUP:

  .itext (TEXT) ALIGN(0x1000): {}

  .itext_vle (VLECODE) ALIGN(0x1000): {

         *(.itext)

         *(.itext_vle)

  }
